★ 주의 깊게 볼 포인트
  • AI · LLM 레이어를 백엔드와 분리한 이유: 프롬프트 = 코드, 모델 교체 가능성, 비용 추적 등이 일반 백엔드와 다른 라이프사이클을 가짐.
  • Phase 2 자산 보존소는 메인 DB와 별도 개념으로 설계: 자유 발화 원문 + 변리사 리뷰 패턴은 영구 보존되어 명세서 AI 학습 데이터로 활용 예정.
  • 데이터 격리 미들웨어가 백엔드의 핵심 보안 레이어: PostgreSQL Row-level Security와 함께 이중 방어.
  • 관리자 콘솔은 변리사 프론트엔드와 별도 앱으로 구성: 사용자분(서비스 운영자) 전용. 권한 분리 명확히.

데이터 흐름의 핵심
  • 변리사 개입은 단 3번 (S0+S1, S5, S7) — 나머지 시간은 모두 자동 진행.
  • S5에서 변리사 행동 로그 보존 — 어떤 답변을 어떻게 수정했는지가 Phase 2 명세서 AI의 학습 데이터.
  • S5 → S6 → S5 반복 가능 — 변리사가 만족할 때까지 추가 질문 라운드 진행.
  • S8에서 Immutable Snapshot — 확정된 Table은 변경 불가, 후속 명세서 작성의 안정적 입력.
03

자유 발화 모드 상세 흐름 — V3 핵심

발명자가 5~30분 자유롭게 말한 내용이 어떻게 Table로 분배되는지의 처리 과정. 이 흐름이 본 MVP를 일반 AI 회의록과 차별화하는 핵심.

1
자유 발화 모드 시작
CLIENT
고객이 화면 상단 「자유 발화 모드」 버튼 클릭 → 마이크 권한 요청 → 녹음 시작.
Browser MediaRecorder API · Audio Permission
2
자유 발화 진행
CLIENT
발명자가 5~30분 자유롭게 설명. 일시정지·재개 가능. 중간에 추가 자료(이미지)도 첨부 가능. 「발화 종료」 버튼으로 종료.
Pause/Resume · Image Attachment · Local Buffer
3
음성 파일 업로드 + 영구 보존
SYSTEM
음성 파일을 스토리지로 업로드 (청크 단위로 안정적 전송). 원본 음성은 영구 보존 — Phase 2 학습 자산.
Chunked Upload · Supabase Storage · Phase 2 Asset Bucket
4
STT 변환
AI
Whisper API로 음성 → 텍스트 변환. 한국어 + 기술 용어 인식. 문장 단위 분리, 시간 코드(timestamp) 보존.
OpenAI Whisper large-v3 · Timestamp Mapping
5
자유 발화 분배 (LLM)
AI
Sonnet 4.7이 STT 텍스트 + Table 스키마 + 필드별 필수 요소 정의를 함께 받아, 발화 내용을 각 필드로 분배. 각 필드 값에 출처(원문 중 어느 부분에서 가져왔는지) 기록.
Claude Sonnet 4.7 · Long Context · Structured Output (JSON)
6
기타 메모 분류
AI
어느 필드에도 명확히 속하지 않는 내용은 「기타 메모」로 별도 보존. 변리사가 S5에서 검토 후 활용.
Fallback Bucket · Attorney Review Hint
7
고객에게 간략 요약 표시
CLIENT
「말씀해주신 내용을 정리했습니다. 핵심 포인트, 효과, 구성 등이 채워졌습니다」 정도의 간략 요약만 노출. 세부 분배 검증은 변리사 단계에서.
Minimal UX · No Validation Burden
8
빈 필드 보충 Q&A 자동 진입
AI
자유 발화 후에도 비어 있는 필드는 자동으로 Q&A 모드로 전환되어 보충 질문 진행. 1회 재질문 정책 동일 적용.
Auto Mode Switch · 1-Retry Policy
9
변리사 검증 도구 노출
ATTORNEY
S5 리뷰 화면에서 변리사가 음성 원문 재생, STT 텍스트 확인, 필드별 출처 하이라이트, 재분배 요청 등 도구 사용 가능.
Audio Player · Source Highlighting · Re-distribute Action
자유 발화 모드의 데이터 흐름 핵심
  • 3중 데이터 보존: 음성 원문 (영구) + STT 텍스트 (영구) + 분배된 Table 값 (변경 가능). 변리사가 언제든 원문으로 돌아갈 수 있음.
  • 발명자 부담 최소화: 분배 결과의 정확도 검증은 발명자에게 요구하지 않음. 간략 요약만 보여주고 변리사가 책임.
  • 빈 필드 자동 보충: 자유 발화로 다 채워지지 않으면 Q&A로 자연스럽게 이어져, 발명자는 모드 전환을 의식하지 않음.
  • Phase 2 핵심 자산: 음성 원문은 정제된 Q&A보다 풍부한 정보·뉘앙스를 담고 있어, 명세서 AI 학습의 가장 가치 있는 데이터.
